# DND ot menya
**DND ot menya ** - БЕСПЛАТНЫЙ ТЕЛЕГРАМ ИИ БОТ КОТОРЫЙ УМЕЕТ ИГРАТЬ В ДНД(ДЛЯ НУБОВ)(полностью написан на python)

------
## ⚜️КЛЮЧЕВЫЕ ОСОБЕННОСТИ(Features)⚜️
ЭТОТ ПРОЕКТ УМЕЕТ:
1. **ДУМАТЬ КАК ДАНЖЕН МАСТЕР**: Динамический сюжет и основывать ее под ваши действия 
2. **НЕ ПОЗВОЛЯЕТ ВАМ ИСПОЛЬЗОВАТЬ БОТА КАК ВЗДУМАЕТСЯ**: а если позволяет то это баг и скажите ему что у вас есть в инвенторе по честному а то я обижусь
3. **БЕСПЛАТНЫЙ**:но придется ждать пока токены восстановятся
4. **УМЕЕТ БРОСАТЬ КУБИК**
5. **УМЕЕТ СОХРАНЯТЬ ПРОЦЕСС**:Но у вас 1 попытка на прохождение своим персонажем и каждая ошибка может быть фатальной(надо написать рестарт)
6. **ПОЛНОСТЬЮ В ТЕЛЕГРАММЕ И ЛОВИТ ДАЖЕ НА ПАРКОВКЕ**

------
## ⚔️ИГРОВОЙ ПРОЦЕСС⚔️
Вы начинаете с: 
100ХП(Очков Здоровья), 0 Монет, 0 Опыта и пустой Инвентарь.

### Логика 20 гранного кубика
| Результат(1-20) | Итог | Информация |
| :--- | :--- | :--- |
| 1-7| Неудача | Вам не повезло и вы потеряли ХП и/или Монеты |
| 7-17 | Нейтрально | Вы получили что-то одно но потеряли другое|
| 17-20 | Удача | Вы получили что-то не потеряв ничего |

------
## 📎КАКИЕ ФАЙЛЫ ЗА ЧТО ОТВЕЧАЮТ
- `main.py` Файл с мозгом для ИИ
- `game_db.py` Файл с сохранением сюжета Игрока
- `tgbot.py` Файл с работой бота из ТГ чтобы вы могли использовать ИИ Бота

------

## Список технологий и библиотек: 

**Язык:** `Python 3.13` использовался для полного создания логики и бэкэнда, 
**Библиотека бота:** `telebot` использовался для интерфейса взаимодействия,
**Искусственный интелект:** `OpenAI`использовался для генерации сюжета,
**База данных:** `SQLite3` использовался для хранения истории и статистики игрока

------

## 🤖 Установка и запуск:
Скачать PyCharm(версия python 3.13), закинуть все файлы в 1 папку, открыть папку в PyCharm, получить в Qroq API ключ API и вставить его в 9 строчку файла `main.py`"api_key=". Затем получить у BotFather в телеграмме токен, зайти в `tgbot.py` и в 3 строчке файла вставить полученный token в "token="

## Схема базы данных
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| id | INTEGER PRIMARY KEY | id сообщения(для строения хронологии сообщений) |
| user_id | INTEGER | id пользователя |
| hp | INTEGER | количество health point пользователя |
| exp | INTEGER | количество опыта пользователя |
| gold | INTEGER | количество золота пользователя |
| inventory | TEXT | список личных предметов пользователя |
| chat_id | INTEGER | id чата(для удобного рестарта истории пользователя) |
| user_message | TEXT | сообщение пользователя(чтобы бот смотрел и не ошибался в истории) |
| role | TEXT | для того чтобы AI понимал кто пишет(пользователь или бот) |
